# 角色人格测试 Skill

这是一个仿造16型人格测试的技能，根据用户指定的剧集创建角色人格测试问卷。

## 使用场景

当用户想要创建类似16型人格的测试题，测试用户更像剧集中的哪个角色时使用此skill。

## 工作流程

### 第一步：获取剧集角色信息

1. 使用 WebSearch 搜索获取角色信息：
   - 搜索 "剧集名称 角色介绍"
   - 搜索 "剧集名称 演员表"

2. 缓存网址网页：
   - 使用 `python_scripts/scrape.py` 将搜索到的网址内容保存到 `tmp/` 目录
   - 命令示例：`python scrape.py 鬼灭之刃 https://www.tvmao.com/drama/XGheaGsk/actors`

3. 对角色姓名进行正则符号分词，形成搜索队列：
   - 使用 `python_scripts/parse_images.py` 中的 `smart_tokenize` 函数
   - 分词策略：
     - 常见分隔符分割（顿号、逗号、空格）
     - 提取末尾2-4个字作为关键词
     - 如jieba可用，使用jieba分词补充
   - 例如："灶门炭治郎" → ["炭治郎", "灶门"]

4. 在本地缓存网页中搜索角色图片：
   - 使用 `python_scripts/parse_images.py` 中的 `find_images_by_keywords` 函数
   - 在缓存HTML中查找包含关键词的图片
   - 匹配策略：优先匹配alt文字，其次匹配父级上下文
   - 取离角色名称最近的img标签的src属性作为角色图片URL
   - 命令示例：`python parse_images.py 鬼灭之刃 灶门炭治郎`

5. 获取6-18个主要角色的人设、性格特点

6. 分析每个角色的人格特征：
   - 性格特点
   - 行为模式
   - 价值观
   - 处理问题的方式

### 第二步：创建项目目录

在项目根目录创建：
```
{剧集名称}/
├── data/
└── tmp/   # 可选，用于缓存网页
```

### 第三步：设计测试题

1. 设计20-25道选择题，每题4个选项
2. 每题选项对应不同角色
3. 问题类型：情境题、价值观题、偏好题
4. 不依赖具体剧情细节，保持问题普遍性

### 第四步：生成H5问卷

1. 创建HTML文件，数据内联到JavaScript中
2. 包含引导页、问卷页、结果页
3. 结果页移动端图片宽度≥50%

## 输出文件

```
{剧集名称}/
├── data/
│   ├── cast.json          # 演职员表（可选）
│   └── questions.json     # 测试题库
├── tmp/                   # 缓存的网页文件
│   └── *.html
└── quiz.html              # H5问卷页面
```

## questions.json 格式

```json
{
  "drama": "剧集名称",
  "title": "角色人格测试",
  "description": "测测你更像剧集中的哪个角色",
  "characters": {
    "角色名": {
      "name": "角色名",
      "actor": "演员名",
      "description": "匹配结果描述",
      "traits": ["特质1", "特质2"],
      "quote": "台词",
      "avatar": "🫀",
      "imageUrl": ""  // 可选，有图片则填URL
    }
  },
  "questions": [
    {
      "id": 1,
      "text": "问题内容",
      "options": [
        {"text": "选项1", "character": "角色A"},
        {"text": "选项2", "character": "角色B"},
        {"text": "选项3", "character": "角色C"},
        {"text": "选项4", "character": "角色D"}
      ]
    }
  ]
}
```

## 图片获取流程

完整的图片获取流程：

1. **搜索网址**：使用 WebSearch 搜索 "剧集名称 演员表"，获取 tvmao 等影视网站链接

2. **缓存网页**：运行 `python scrape.py <剧集名> <URL>` 将页面保存到 tmp/

3. **分词处理**：运行 `python parse_images.py <剧集名> <角色名>` 
   - 自动对角色名进行正则符号分词
   - 例如："灶门炭治郎" → ["炭治郎", "灶门"]

4. **搜索图片**：脚本会在缓存网页中查找离角色名最近的 img 标签

如需获取角色图片：
1. 搜索电视猫等影视网站
2. 使用上述流程自动获取

## H5实现要点

1. **数据必须内联** - 不要用fetch加载外部JSON
2. **首页3D球面动画** - 斐波那契球面分布
3. **随机效果** - 每个角色的动画延迟和持续时间随机，保持动画效果的多样性
4. **移动端适配** - 结果页图片宽度≥50%
